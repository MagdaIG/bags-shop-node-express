API REST Documentation

Documentación completa de endpoints, métodos HTTP y ejemplos de uso

Base URL: http://localhost:3000/api

Información General de la API

Características
  • API RESTful completa
  • Validaciones robustas
  • Manejo de errores consistente
  • Transacciones para operaciones críticas
  • Respuestas en formato JSON
Autenticación

Actualmente la API no requiere autenticación, pero incluye:

  • Validación de datos de entrada
  • Contraseñas hasheadas con bcrypt
  • Validación de email único

Usuarios API

POST
/api/usuarios
201 Created

Crear un nuevo usuario

Request Body:
{ "nombre": "Juan Pérez", "email": "juan@example.com", "password": "MiPassword123" }
Validaciones:
  • nombre: 2-60 caracteres, requerido
  • email: formato válido, único en la BD
  • password: mínimo 8 caracteres, al menos 1 número
Response:
Success (201):
{"success": true, "message": "Usuario creado exitosamente", "data": {...}}

Error (400):
{"success": false, "message": "Error de validación", "errors": [...]}
GET
/api/usuarios
200 OK

Obtener todos los usuarios

Response:
{ "success": true, "data": [ { "id": 1, "nombre": "Juan Pérez", "email": "juan@example.com", "createdAt": "2024-01-01T00:00:00.000Z", "updatedAt": "2024-01-01T00:00:00.000Z" } ] }
GET
/api/usuarios/:id
200 OK

Obtener un usuario específico por ID

Parameters:
  • id: ID del usuario (integer)
Response:
Success (200):
{"success": true, "data": {...}}

Error (404):
{"success": false, "message": "Usuario no encontrado"}
PUT
/api/usuarios/:id
200 OK

Actualizar un usuario existente

Request Body:
{ "nombre": "Juan Carlos Pérez", "email": "juancarlos@example.com" }
Response:
Success (200):
{"success": true, "message": "Usuario actualizado exitosamente", "data": {...}}
DELETE
/api/usuarios/:id
200 OK

Eliminar un usuario

Response:
Success (200):
{"success": true, "message": "Usuario eliminado exitosamente"}

Pedidos API

POST
/api/pedidos
201 Created

Crear un nuevo pedido (con transacción)

Request Body:
{ "usuario_id": 1, "producto": "Makeup Bag Lavender", "cantidad": 2 }
Validaciones:
  • usuario_id: debe existir en la BD
  • producto: 1-100 caracteres, requerido
  • cantidad: número entero mayor a 0
Transacciones:

Este endpoint utiliza transacciones para garantizar la consistencia de datos. Si falla la creación del pedido, se hace rollback automático.

Response:
Success (201):
{"success": true, "message": "Pedido creado exitosamente", "data": {...}}

Error (400):
{"success": false, "message": "Error de validación", "errors": [...]}
GET
/api/pedidos
200 OK

Obtener todos los pedidos

Response:
{ "success": true, "data": [ { "id": 1, "usuario_id": 1, "producto": "Makeup Bag Lavender", "cantidad": 2, "fecha_pedido": "2024-01-01", "usuario": { "id": 1, "nombre": "Juan Pérez", "email": "juan@example.com" } } ] }
GET
/api/pedidos/usuario/:id
200 OK

Obtener pedidos de un usuario específico

Parameters:
  • id: ID del usuario (integer)
Response:
Success (200):
{"success": true, "data": [...]}

Error (404):
{"success": false, "message": "Usuario no encontrado"}

Ejemplos de Uso con cURL

Crear Usuario
curl -X POST http://localhost:3000/api/usuarios \ -H "Content-Type: application/json" \ -d '{ "nombre": "María García", "email": "maria@example.com", "password": "MiPassword123" }'
Crear Pedido
curl -X POST http://localhost:3000/api/pedidos \ -H "Content-Type: application/json" \ -d '{ "usuario_id": 1, "producto": "Makeup Bag Lavender", "cantidad": 2 }'
Obtener Usuarios
curl http://localhost:3000/api/usuarios
Obtener Pedidos
curl http://localhost:3000/api/pedidos

Manejo de Errores

Códigos de Estado HTTP
  • 200 - OK (Operación exitosa)
  • 201 - Created (Recurso creado)
  • 400 - Bad Request (Error de validación)
  • 404 - Not Found (Recurso no encontrado)
  • 500 - Internal Server Error
Formato de Respuesta de Error
{ "success": false, "message": "Descripción del error", "errors": [ "Detalle específico del error 1", "Detalle específico del error 2" ] }
